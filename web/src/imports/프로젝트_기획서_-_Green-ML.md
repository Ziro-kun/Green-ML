# 🌿 Green-ML (AI 탄소 가계부 & 튜너)

## 1. 프로젝트 개요

- **배경**: AI 모델 학습 시 발생하는 전력 소비와 탄소 배출 문제에 대한 인식 제고 필요
- **목표**: AI 모델 학습 시 소비되는 에너지 및 탄소 배출량을 직관적으로 보여주고, 이를 줄일 수 있는 최적화 가이드를 제공하는 모바일 앱 프로토타입 개발
- **슬로건**: "지구를 살리는 스마트한 한 줄의 코드, Green-ML"

---

## 2. 핵심 타겟 및 프로토타입 목표

- **타겟 유저**: 딥러닝을 학습하는 학생, 토이 프로젝트를 진행하는 주니어 개발자
- **MVP 핵심 가치**:
  - **빠른 시각화**: 복잡한 설정 없이 코드 몇 줄 추가로 내 모델의 탄소 배출량 확인
  - **실용적 대안**: 당장 적용 가능한 수준의 최적화 팁(Batch size 조정, Mixed Precision 등) 제공
- **프로토타입 제약**: 짧은 기간 내 집중적인 개발을 위해 핵심 기능 위주 2~3개 페이지로 압축하여 구현

---

## 3. 주요 기능 (MVP - 단일 흐름 내 2~3 Pages)

### 📄 Page 1: 탄소 영수증 대시보드 (Home/Dashboard)

- **실시간 모니터링 요약**: 현재 진행 중인 학습 세션의 예상 탄소 배출량 및 소비 전력을 대시보드 형태로 제공
- **직관적 환산 지표**: 소비 전력을 "소나무 N그루 심기", "자동차 N km 주행" 등 알기 쉬운 지표로 변환하여 '탄소 영수증' UI 형태로 시각화 제공

### 💡 Page 2: 최적화 어드바이저 (Optimization Guide)

- **간편 최적화 팁 제공**: 에너지 효율을 높일 수 있는 구체적이고 즉시 적용 가능한 가이드 제시 (예: "Mixed Precision(FP16)을 적용하면 약 30% 에너지를 절약할 수 있습니다.")
- **시뮬레이션 비교**: 최적화 기법 적용 전후의 예상 탄소 절감 효과를 간단한 차트나 수치 박스로 나란히 비교

### ⚙️ Page 3: 기록 및 설정 (History & Settings)

- **간편 기록 확인**: 과거 완료된 테스트/학습 세션들의 탄소 배출 이력을 단순 리스트 형태로 조회
- **상태 관리**: 현재 API 서버(로컬/원격) 연결 상태 확인 및 앱 기본 환경 설정

## 4. 시스템 아키텍처 및 데이터 수집 흐름 (Architecture & Data Flow)

_본 모바일 앱(React Native)은 데이터를 직접 측정하지 않으며, 실제 측정이 이루어지는 AI 학습 환경(Python)으로부터 백엔드를 통해 데이터를 전달받아 시각화하는 역할을 합니다._

1. **AI 학습 환경 (데이터 수집 / Python)**
   - 사용자의 기존 AI 학습 스크립트(GPU 서버, PC, Colab 등)에 배출량 측정을 위한 `CodeCarbon` 관련 코드를 3~4줄만 추가하여 실행합니다.
   - 학습을 진행하면서 측정된 탄소 배출량과 전력 소비량 데이터를 백엔드로 전송합니다.
2. **FastAPI 백엔드 (데이터 스토어 / Python)**
   - 학습 환경에서 쏴주는 데이터를 HTTP API(예: POST /record)로 받아 SQLite DB에 저장합니다.
3. **모바일 앱 (데이터 시각화 / React Native)**
   - 앱이 백엔드와 통신하여 데이터를 조회하고, 폰 화면에 예쁜 '탄소 영수증'과 최적화 가이드를 띄워줍니다. (앱 자체의 연산량은 매우 가볍습니다)

## 5. 기술 스택

| **분야**         | **기술/도구**         | **선정 이유**                                                                                       |
| ---------------- | --------------------- | --------------------------------------------------------------------------------------------------- |
| **Frontend**     | React Native (Expo)   | 학습 내용을 바로 적용 가능, 빠른 UI 컴포넌트 개발 및 iOS/Android 통합 테스트 용이                   |
| **Backend**      | FastAPI (Python)      | 가볍고 빠른 API 구축 가능, AI 측정 라이브러리(Python)와의 완벽한 호환성                             |
| **Database**     | SQLite (+ SQLAlchemy) | 복잡한 DB/시계열 서버 인프라 구축 생략, 백엔드와 통합된 로컬 파일형 DB로 즉각적인 저장 및 조회 구현 |
| **Measurement**  | CodeCarbon            | 기존 ML 코드에 데코레이터/코드 몇 줄만 추가하여 빠르고 쉽게 GPU/CPU 탄소 배출량 측정 가능           |
| **Infra / Test** | Local Server + Ngrok  | 별도의 클라우드 배포(Docker 등) 없이 개발 PC에서 Ngrok으로 터널링하여 모바일과 즉시 API 연동        |

---

## 6. 단계별 MVP 개발 전략

1. **1단계 (Backend+DB)**: CodeCarbon을 활용해 더미 ML 스크립트에서 측정값을 추출하고, FastAPI와 SQLite를 통해 데이터를 저장/제공하는 API (GET/POST) 구현
2. **2단계 (Frontend)**: React Native (Expo)를 활용해 대시보드와 어드바이저 2개 핵심 화면 UI 구성 및 네비게이션 적용
3. **3단계 (Integration)**: Ngrok을 통해 외부 개방된 로컬 FastAPI 주소를 RN 앱과 연동하고, 데이터 시각화 결과 최종 확인
